# План реализации — "Попутчик ИТМО"

Статусы: `в работе`, `не начато`, `готово`.

После завершения каждого шага: обновить статус здесь, обновить
`current_state.md`, при необходимости — `architecture.md` /
`design_system.md`.

---

## 1. Инициализация — **готово**

Создание `ai_context/`, `CLAUDE.md`, инициализация Vite + React + TS,
подключение Tailwind + shadcn/ui + Telegram Mini Apps SDK + Supabase JS,
базовая структура папок, `.env.example`. Без бизнес-логики и экранов.

## 2. БД и Supabase — **готово**

Миграции применены к Supabase-проекту через Management API: таблицы
`users`, `locations` (единая таблица точек вместо `campuses`/`dormitories`),
`routes`, `vehicles`, `driver_documents`, `moderation_requests`, `trips`,
`trip_members`, `trip_requests`; enum-типы; RLS включён с явными политиками
на всех таблицах; приватный Storage-bucket `driver-documents` с политиками.
Справочники засидированы (5 корпусов, 5 общежитий, 50 маршрутов). Детали —
в `current_state.md` и `architecture.md` (раздел 5.2).

## 3. Telegram-авторизация, регистрация (номер ИСУ) и мок ITMO ID — **в работе**

Разбит на части:

- **3a — готово**: весь UI-флоу на мок-провайдере авторизации (экраны
  Splash 6.1, регистрация ИСУ 6.2, мок-вход ITMO ID, заглушка главного
  экрана), `services/itmoId.ts`, `services/auth.ts` (интерфейс +
  мок-реализация), `AuthProvider`/`useAuth`/`AppFlow`. Детали —
  `current_state.md`.
- **3b — в работе (следующее)**: Edge Function `telegram-auth` (проверка
  HMAC initData + Admin API), Supabase-реализация `AuthBackend`, запись
  `public.users`, секреты Edge Function. Стратегия — `architecture.md` 5.1.

## 4. Роли и профиль — не начато

Профиль пользователя, базовые роли (пассажир/водитель/админ), экран
профиля.

## 5. Маршруты — не начато

UI и логика работы со справочниками маршрутов/корпусов/общежитий, выбор
маршрута при создании поездки.

## 6. Создание и поиск поездок — не начато

Создание поездки водителем, поиск/фильтрация поездок пассажиром, заявки на
участие (`trip_requests`), подтверждение участников (`trip_members`).

## 7. Регистрация и верификация водителя — не начато

Подача заявки на роль водителя, данные автомобиля (`vehicles`), загрузка
документов (`driver_documents`), статусы верификации.

## 8. Админ-панель модерации — не начато

Интерфейс для администраторов: рассмотрение `moderation_requests` и
`driver_documents`, управление пользователями.

## 9. Поездки без водителя (release 2) — не начато

Сценарий координации совместной поездки без привязки к конкретному
водителю/автомобилю.

## 10. UI/UX полировка (dark/light, Telegram themes) — не начато

Адаптация под Telegram `themeParams`, light/dark режимы, финальная
полировка интерфейса по `design_system.md`.

## 11. Тестирование и деплой — не начато

Настройка тестов, деплой frontend на Vercel, проверка интеграций.

## 12. Документация — не начато

Финальная документация проекта (README, инструкции по запуску/деплою).
